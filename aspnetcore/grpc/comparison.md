## <a name="grpc-strengths"></a><span data-ttu-id="1e570-138">gRPC 的強項</span><span class="sxs-lookup"><span data-stu-id="1e570-138">gRPC strengths</span></span>

### <a name="performance"></a><span data-ttu-id="1e570-139">效能</span><span class="sxs-lookup"><span data-stu-id="1e570-139">Performance</span></span>

<span data-ttu-id="1e570-140">gRPC 訊息是使用 [Protobuf](https://developers.google.com/protocol-buffers/docs/overview)（有效率的二進位訊息格式）進行序列化。</span><span class="sxs-lookup"><span data-stu-id="1e570-140">gRPC messages are serialized using [Protobuf](https://developers.google.com/protocol-buffers/docs/overview), an efficient binary message format.</span></span> <span data-ttu-id="1e570-141">Protobuf 會在伺服器和用戶端上迅速序列化。</span><span class="sxs-lookup"><span data-stu-id="1e570-141">Protobuf serializes very quickly on the server and client.</span></span> <span data-ttu-id="1e570-142">Protobuf 序列化會產生小型訊息承載，在有限的頻寬案例（例如行動應用程式）中很重要。</span><span class="sxs-lookup"><span data-stu-id="1e570-142">Protobuf serialization results in small message payloads, important in limited bandwidth scenarios like mobile apps.</span></span>

<span data-ttu-id="1e570-143">gRPC 是針對 HTTP/2 設計的，這是 HTTP 的主要版本，可透過 HTTP 1.x 提供顯著的效能優勢：</span><span class="sxs-lookup"><span data-stu-id="1e570-143">gRPC is designed for HTTP/2, a major revision of HTTP that provides significant performance benefits over HTTP 1.x:</span></span>

### <a name="not-human-readable"></a><span data-ttu-id="1e570-202">不是人類看得懂的</span><span class="sxs-lookup"><span data-stu-id="1e570-202">Not human readable</span></span>

<span data-ttu-id="1e570-208">[伺服器反映](https://github.com/grpc/grpc/blob/master/doc/server-reflection.md)和[gRPC 命令列工具](https://github.com/grpc/grpc/blob/master/doc/command_line_tool.md)等功能，可協助您使用二進位 Protobuf 訊息。</span><span class="sxs-lookup"><span data-stu-id="1e570-208">Features such as [server reflection](https://github.com/grpc/grpc/blob/master/doc/server-reflection.md) and the [gRPC command line tool](https://github.com/grpc/grpc/blob/master/doc/command_line_tool.md) exist to assist with binary Protobuf messages.</span></span> <span data-ttu-id="1e570-209">此外，Protobuf 訊息支援在 [JSON 之間進行轉換](https://developers.google.com/protocol-buffers/docs/proto3#json)。</span><span class="sxs-lookup"><span data-stu-id="1e570-209">Also, Protobuf messages support [conversion to and from JSON](https://developers.google.com/protocol-buffers/docs/proto3#json).</span></span> <span data-ttu-id="1e570-210">內建的 JSON 轉換可讓您有效率地在進行偵錯工具時，將 Protobuf 訊息轉換成人類可讀取的表單。</span><span class="sxs-lookup"><span data-stu-id="1e570-210">The built-in JSON conversion provides an efficient way to convert Protobuf messages to and from human readable form when debugging.</span></span>
